¿Qué hace la simulación?

La simulación representa visualmente el programa creado con bloques.

Sirve para:

  • visualizar movimientos
  • comprobar la secuencia del programa
  • detectar errores básicos
  • entender el funcionamiento antes de usar el robot real

La simulación es una representación simplificada. No sustituye al robot real ni garantiza por sí sola que el movimiento sea alcanzable o seguro en la celda real.

¿Qué se compara con el robot real?

Se compara:

  • el plan generado por la interfaz
  • con los puntos reales alcanzados por el robot

La comparación solo es válida si los datos reales pertenecen a la misma versión del programa.

La versión se identifica mediante el snapshot del programa y su ID corto de telemetría.

Guía rápida de uso

Simulación

  1. Crea el programa en la interfaz principal.
  2. Guarda el programa si quieres conservarlo.
  3. Pulsa "Simular".
  4. Revisa la trayectoria y la secuencia.

Robot real

  1. Activa la instrumentación en la interfaz principal.
  2. Genera el programa instrumentado.
  3. Comprueba que MQTT está conectado al broker/topic correcto.
  4. Ejecuta el programa en el robot real.
  5. La interfaz recibirá y guardará las ejecuciones reales compatibles.
  6. Pulsa "Comparar última ejecución con el plan actual".

Las ejecuciones reales pueden recibirse y compararse más tarde, siempre que exista el plan/snapshot correspondiente.

Importante
  • Para comparar no es obligatorio simular, pero sí debe existir un plan generado del programa.
  • Si haces cambios en el programa, se genera una nueva versión.
  • El nombre del programa no identifica la versión exacta.
  • La comparación se basa en el snapshot y en el ID corto de telemetría.
  • Si el robot envía datos de otro snapshot, la interfaz no debe mezclarlos con el plan actual.
